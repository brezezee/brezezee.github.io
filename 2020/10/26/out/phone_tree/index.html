Today’s 4th grade field trip has been cancelled, and the parents are in a panic. They have been trying to
put together a phone tree to make sure that all the parents get this critical update, but the evil Harold 1
Hacker (who has some strange sort of grudge with 4th graders) has unleashed a program that foils the parents’ attempts. Every time they come up with a phone tree and send it to the principal via email, Harold Hacker’s program attempts to maliciously modify the phone tree. The principal can’t figure out who should be called first, and whether or not the call plan will get the information to every parent. It’s pandemonium.
This is where you come in. Your job is to write a program that will read in the proposed phone tree and determine whether or not the call plan can be used to reach every parent. If it can, you will tell the principal who to call first and how long it will take for every parent to be called. If it can’t, we will simply curse the devilish Harold Hacker. He’s the worst!
The basic idea is simple: the principal will call one parent, who in turn will call the next parent, who in turn will call the next, and so on. The phone tree proposal begins with the total number of parents who need to be called. Every subsequent line lists the name of one parent, the time it takes that parent to make the call, and the next parent who that parent has been assigned to call. An example phone tree is given below (// comments are provided for the reader’s convenience and are not present in the actual data files):
5 // Five parents in this file
Usha 5 Kurt // Usha has been assigned to call Kurt.
Kurt 2 Usha // is a fast dialer. It only takes him 2 seconds to dial!
Leopold 3 Shruti
Shruti 2 Anderson
Anderson 4 Usha
